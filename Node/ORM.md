## ORM
- Significado: Object Relational Mapping
- Ideia: Mapear o que temos no nosso banco de dados, como por exemplo tabelas, em objetos
- Cada ORM terá sua própria sintaxe (é o que muda)
- Alguns ORMs usam decorators (Ex: @Entity) para dizer o que é cada coisa no banco.
- Ex: Prisma: Possui Migrations de forma automatizada, além de possuir integração com TS
